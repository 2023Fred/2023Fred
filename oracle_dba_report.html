Recomendações para Gestores e CIOs

1. Ampliar a FRA — A FRA está com 74,6% de ocupação após as operações de laboratório. Em produção, recomenda-se manter abaixo de 70%. Aumentar para 20 GB via ALTER SYSTEM SET db_recovery_file_dest_size=20G.

2. Agendar DELETE OBSOLETE — Criar job RMAN semanal para remover backups expirados: DELETE OBSOLETE. Isso libera espaço na FRA automaticamente respeitando a política de retenção de 7 dias.

3. Algoritmo de Compressão — Com Advanced Compression Option licenciada, adotar MEDIUM para backups diários (melhor equilíbrio CPU/espaço). Usar HIGH apenas em janelas de backup noturnas onde CPU não é crítica.

4. Teste de Recovery Periódico — Executar RESTORE DATABASE VALIDATE mensalmente para garantir a integridade dos backupsets sem alterar o banco produtivo.

5. Disaster Recovery (DR) — O próximo passo natural é configurar Oracle Data Guard (standby físico) ou replicar os backups para storage offsite, garantindo RTO < 15 min e RPO < 1h em cenários de desastre total no datacenter.
