Demonstração ACID Oracle

Executada no PDB_APP1 com tabela pdbadmin.acid_teste e pdbadmin.vendas_teste.

A — Atomicidade

-- Transferência de R$300: ambas as operações em uma transação
UPDATE acid_teste SET valor = valor - 300 WHERE id = 1;  -- 1000 → 700
UPDATE acid_teste SET valor = valor + 300 WHERE id = 2;  --  500 → 800
COMMIT;

Resultado: COMMIT = ambas persistem | ROLLBACK = nenhuma muda
Verificado: ID=1: R$700 | ID=2: R$800 ✓

C — Consistência

-- Violação de PK → ROLLBACK automático
INSERT INTO acid_teste VALUES (1, 999, 'ID duplicado');
-- ORA-00001: unique constraint violated

-- Violação de CHECK → ROLLBACK automático
UPDATE acid_teste SET valor = -100 WHERE id = 1;
-- ORA-02290: check constraint (valor >= 0) violated

Resultado: Banco permaneceu consistente em ambos os casos ✓

I — Isolamento (MVCC)

SELECT ora_rowscn FROM acid_teste WHERE rownum=1;
-- SCN: 2.378.451
-- Reader NÃO bloqueia Writer
-- Writer NÃO bloqueia Reader
-- Oracle usa Undo Segments para leitura consistente multiversão

Resultado: Isolamento READ COMMITTED padrão; sem deadlock em sessões concorrentes ✓

D — Durabilidade

SELECT count(*) FROM v$archived_log
WHERE first_time > SYSDATE - 1/24 AND standby_dest='NO';
-- 15 archived logs = cadeia de durabilidade confirmada
-- COMMIT → Redo Buffer → LGWR → Redo Log → Archiver → FRA

Resultado: 15 archived logs protegem dados commitados ✓
